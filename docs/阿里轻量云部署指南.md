# 阿里轻量云部署指南

本文档记录 FastAPI + SQLite + 本地文件工作区版本的部署方式。代码放在 GitHub，运行数据放在服务器的 `/data/ProjectWorkspace`，不要把 `app.sqlite`、PDF、项目文件提交到 GitHub。

## 1. 服务器准备

推荐镜像：Ubuntu 22.04 或 24.04。

在阿里云轻量应用服务器控制台开放防火墙端口：

```text
22    SSH
80    HTTP
443   HTTPS
```

登录服务器：

```bash
ssh root@your-server-ip
```

安装基础依赖：

```bash
apt update
apt install -y python3 python3-venv python3-pip git nginx sqlite3
```

## 2. 拉取代码

```bash
mkdir -p /opt
cd /opt
git clone https://github.com/silicontalks/cihlab-wiki-v1.git
cd /opt/cihlab-wiki-v1/backend
```

创建虚拟环境并安装依赖：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 3. 准备数据目录

```bash
mkdir -p /data/ProjectWorkspace
```

应用通过环境变量读取工作区路径：

```text
PROJECT_WORKSPACE_PATH=/data/ProjectWorkspace
```

该目录会保存：

```text
/data/ProjectWorkspace/app.sqlite
/data/ProjectWorkspace/projects/
/data/ProjectWorkspace/backups/
/data/ProjectWorkspace/attachments/
```

## 4. 手动启动测试

```bash
cd /opt/cihlab-wiki-v1/backend
source .venv/bin/activate
PROJECT_WORKSPACE_PATH=/data/ProjectWorkspace \
  uvicorn app.main:app --host 0.0.0.0 --port 8000
```

测试：

```text
http://your-server-ip:8000/api/health
```

预期返回：

```json
{"status":"ok"}
```

## 5. systemd 服务

创建 `/etc/systemd/system/cihlab-wiki.service`：

```ini
[Unit]
Description=CIHLab Wiki FastAPI App
After=network.target

[Service]
WorkingDirectory=/opt/cihlab-wiki-v1/backend
Environment=PROJECT_WORKSPACE_PATH=/data/ProjectWorkspace
ExecStart=/opt/cihlab-wiki-v1/backend/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
systemctl daemon-reload
systemctl enable cihlab-wiki
systemctl start cihlab-wiki
systemctl status cihlab-wiki
```

查看日志：

```bash
journalctl -u cihlab-wiki -f
```

## 6. 登录账号申请与创建

当前版本没有开放 Web 自助注册入口。新用户需要向管理员申请账号，由管理员登录服务器后在 SQLite 数据库里创建账号、分配项目权限。

默认管理员账号会在首次初始化数据库时自动创建：

```text
账号：admin
密码：admin
```

生产环境上线后必须先修改默认密码：

```bash
sqlite3 /data/ProjectWorkspace/app.sqlite
```

进入 SQLite 后执行：

```sql
UPDATE app_users
SET password = 'your-strong-password', updated_at = datetime('now')
WHERE username = 'admin';
```

新增普通账号：

```sql
INSERT INTO app_users (username, password, display_name, is_admin, created_at, updated_at)
VALUES ('user1', 'password1', '用户1', 0, datetime('now'), datetime('now'));
```

查看项目 ID：

```sql
SELECT id, name, created_by FROM projects WHERE deleted_at IS NULL;
```

授权普通账号访问某个项目：

```sql
INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 5, datetime('now'));
```

取消项目授权：

```sql
DELETE FROM project_access
WHERE username = 'user1' AND project_id = 5;
```

退出 SQLite：

```sql
.quit
```

权限规则：

- `is_admin = 1` 的管理员账号可以访问所有项目。
- 普通账号可以访问自己创建的项目。
- 普通账号也可以访问 `project_access` 表里额外授权的项目。

目前密码是明文保存在 `app_users.password` 字段里，因此不要复用个人常用密码。后续如果面向更多用户，应升级为哈希密码存储和后台账号管理页面。

## 7. Nginx 反向代理

后端已经从 GitHub 拉取部署到服务器后，推荐让 Uvicorn 只监听本机 `127.0.0.1:8000`，公网统一通过 Nginx 的 80/443 端口访问。

### 7.1 同域名部署

如果前端页面和 API 都由这台 FastAPI 服务提供，使用下面配置即可。浏览器访问域名根路径 `/`，API 走同一个域名下的 `/api/*`，登录 Cookie 不会有跨域问题。

创建 `/etc/nginx/sites-available/cihlab-wiki`：

```nginx
server {
    listen 80;
    server_name your-domain-or-server-ip;

    client_max_body_size 100M;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/cihlab-wiki /etc/nginx/sites-enabled/cihlab-wiki
nginx -t
systemctl reload nginx
```

访问：

```text
http://your-domain-or-server-ip/
http://your-domain-or-server-ip/api/health
```

### 7.2 GitHub Pages 或其他前端域名反代到后端

如果静态前端已经部署在 GitHub Pages，而后端部署在阿里云，需要注意：GitHub Pages 本身不能把 `/api` 反向代理到阿里云后端。可选做法有两种：

1. 推荐：让用户直接访问阿里云域名，由 Nginx 同时代理页面和 API，也就是使用 7.1 的同域名部署。
2. 如果必须保留 GitHub Pages 前端，需要给阿里云后端单独准备一个 API 域名，例如 `api.example.com`，并在前端代码里把 API 基础地址改成这个域名。同时后端需要允许 GitHub Pages 域名跨域请求。

API 独立域名的 Nginx 示例：

```nginx
server {
    listen 80;
    server_name api.example.com;

    client_max_body_size 100M;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果走跨域 API 域名，当前代码还需要同步调整：

- `backend/app/main.py` 的 `CORSMiddleware.allow_origins` 增加 GitHub Pages 前端域名，例如 `https://silicontalks.github.io`。
- 前端请求不能继续默认打到当前域名的 `/api`，需要支持配置 `API_BASE_URL=https://api.example.com`。
- 登录 Cookie 跨站使用时还需要设置 `SameSite=None; Secure`，并启用 HTTPS。否则浏览器可能不会携带 `session_token`。

所以在当前实现下，最省事、最稳定的生产方案是同域名部署：用户访问阿里云域名，Nginx 反代到本机 Uvicorn。

## 8. HTTPS

如果已经有域名并完成解析，可以安装证书：

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx
```

中国大陆服务器绑定域名通常需要先完成备案。没有备案时，可以先用服务器 IP 访问，或使用中国香港等不需要大陆备案的地域。

## 9. 更新部署

```bash
cd /opt/cihlab-wiki-v1
git pull
cd backend
source .venv/bin/activate
pip install -r requirements.txt
systemctl restart cihlab-wiki
```

## 10. 备份建议

至少备份以下目录：

```text
/data/ProjectWorkspace/app.sqlite
/data/ProjectWorkspace/projects/
/data/ProjectWorkspace/backups/
```

建议同时使用：

- 应用内备份：`/api/workspace/backup`
- 服务器快照或数据盘快照
- 定期把 `/data/ProjectWorkspace` 同步到另一台机器或对象存储
