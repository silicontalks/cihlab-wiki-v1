# 服务器账号与项目授权指南

本文档记录如何在服务器上的 SQLite 数据库里新增登录账号，并授权该账号访问指定项目。

当前版本没有 Web 自助注册和后台账号管理页面。账号、密码、项目权限都需要管理员在服务器上手动维护。

## 1. 登录服务器

```bash
ssh your-user@your-server-ip
```

例如：

```bash
ssh ubuntu@122.51.36.237
```

如果你是用 `root` 登录，则命令类似：

```bash
ssh root@122.51.36.237
```

## 2. 打开 SQLite 数据库

服务器生产数据默认放在：

```text
/data/ProjectWorkspace/app.sqlite
```

打开数据库：

```bash
sqlite3 /data/ProjectWorkspace/app.sqlite
```

如果提示权限不足，使用：

```bash
sudo sqlite3 /data/ProjectWorkspace/app.sqlite
```

进入成功后会看到：

```text
sqlite>
```

建议先打开表头和列模式，方便查看结果：

```sql
.headers on
.mode column
```

## 3. 查看已有账号

```sql
SELECT username, display_name, is_admin, deleted_at
FROM app_users;
```

字段说明：

| 字段 | 说明 |
|---|---|
| `username` | 登录账号 |
| `display_name` | 页面显示名 |
| `is_admin` | 是否管理员，`1` 是管理员，`0` 是普通用户 |
| `deleted_at` | 删除时间，非空通常表示已删除 |

管理员账号 `is_admin = 1` 可以访问所有项目。普通账号只能访问自己创建的项目，以及 `project_access` 表里额外授权的项目。

## 4. 查看项目 ID

授权前需要先知道项目 ID：

```sql
SELECT id, name, created_by
FROM projects
WHERE deleted_at IS NULL;
```

输出示例：

```text
id  name      created_by
--  --------  ----------
1   项目A     admin
2   项目B     admin
5   项目C     admin
```

这里第一列 `id` 就是授权时要用的项目 ID。

## 5. 新增普通用户

示例：新增账号 `user1`，密码 `password1`，显示名 `用户1`。

```sql
INSERT INTO app_users (username, password, display_name, is_admin, created_at, updated_at)
VALUES ('user1', 'password1', '用户1', 0, datetime('now'), datetime('now'));
```

注意：

- `is_admin = 0` 表示普通用户。
- `is_admin = 1` 表示管理员，可以访问所有项目。
- 当前密码是明文保存在 `app_users.password` 字段里，不要使用个人常用密码。

## 6. 授权用户访问项目

示例：授权 `user1` 访问项目 ID 为 `5` 的项目。

```sql
INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 5, datetime('now'));
```

如果要授权多个项目，可以多执行几条：

```sql
INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 1, datetime('now'));

INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 2, datetime('now'));

INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 5, datetime('now'));
```

## 7. 检查授权结果

查看某个用户被授权的项目：

```sql
SELECT u.username, u.display_name, p.id AS project_id, p.name AS project_name
FROM project_access pa
JOIN app_users u ON u.username = pa.username
JOIN projects p ON p.id = pa.project_id
WHERE u.username = 'user1';
```

也可以直接查看授权表：

```sql
SELECT username, project_id, created_at
FROM project_access
WHERE username = 'user1';
```

## 8. 取消项目授权

示例：取消 `user1` 对项目 ID `5` 的访问权限。

```sql
DELETE FROM project_access
WHERE username = 'user1' AND project_id = 5;
```

## 9. 修改用户密码

示例：把 `user1` 的密码改为 `new-password`。

```sql
UPDATE app_users
SET password = 'new-password', updated_at = datetime('now')
WHERE username = 'user1';
```

生产环境上线后，必须先修改默认管理员密码：

```sql
UPDATE app_users
SET password = 'your-strong-password', updated_at = datetime('now')
WHERE username = 'admin';
```

## 10. 禁用用户

如果暂时不想让某个账号继续登录，可以软删除：

```sql
UPDATE app_users
SET deleted_at = datetime('now'), updated_at = datetime('now')
WHERE username = 'user1';
```

如果要恢复账号：

```sql
UPDATE app_users
SET deleted_at = NULL, updated_at = datetime('now')
WHERE username = 'user1';
```

## 11. 常见问题

### 11.1 检查授权结果什么也不显示

通常是以下原因之一：

```text
1. 用户名写错了
2. 项目 ID 写错了
3. 授权语句没有执行成功
4. 项目已经被软删除，deleted_at 不是 NULL
```

按顺序检查：

```sql
SELECT username, display_name, is_admin, deleted_at
FROM app_users
WHERE username = 'user1';
```

```sql
SELECT id, name, created_by, deleted_at
FROM projects
WHERE id = 5;
```

```sql
SELECT username, project_id, created_at
FROM project_access
WHERE username = 'user1';
```

如果 `project_access` 里没有记录，重新授权：

```sql
INSERT OR IGNORE INTO project_access (username, project_id, created_at)
VALUES ('user1', 5, datetime('now'));
```

### 11.2 新用户登录后看不到项目

先确认该用户不是管理员：

```sql
SELECT username, is_admin
FROM app_users
WHERE username = 'user1';
```

如果 `is_admin = 0`，继续确认项目授权：

```sql
SELECT username, project_id
FROM project_access
WHERE username = 'user1';
```

如果授权表里有记录，但页面仍看不到项目，确认项目没有被删除：

```sql
SELECT id, name, deleted_at
FROM projects
WHERE id IN (
    SELECT project_id
    FROM project_access
    WHERE username = 'user1'
);
```

## 12. 退出 SQLite

```sql
.quit
```
