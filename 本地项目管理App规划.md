# 本地项目管理 Web App 规划

## 1. 初始目标

先实现一个本地优先的项目管理 Web App，用于管理多个科研、工程或合作项目的进度、文件、论文、专利、周会记录、测试记录和项目文档。

App 通过浏览器访问，本机运行 FastAPI 服务。项目数据和项目文件保存在用户选择的本地工作区中，这个工作区可以放入同步软件目录，从而在 macOS、Windows、Linux 等不同操作系统之间同步使用。

第一阶段采用：

- 本地 Web App
- 后端：FastAPI
- 前端：React + TypeScript
- SQLite 本地数据库
- 每个项目对应一个独立目录
- 项目文件以真实文件形式保存在项目目录下
- SQLite 负责保存项目元信息、标签、文件索引、节点提醒、论文、专利等结构化数据

运行方式：

```text
启动 FastAPI 服务
  -> 浏览器访问 http://127.0.0.1:8000
  -> 读取本地 SQLite
  -> 管理本地项目文件夹
```

核心原则：

- 文件实际保存在本地目录中，方便人工备份、迁移和查看。
- 数据库只保存索引、状态、说明、负责人、时间等结构化信息。
- 先保证单机本地 Web App 可用，后续再扩展多人协作和在线数据库读取。
- 跨平台同步先交给外部同步软件处理，App 自身第一版不实现云同步。
- 所有文件路径在数据库中保存为相对路径，避免不同操作系统下绝对路径不一致。

App 主界面分为左侧项目区和右侧项目详情区。

## 3. 左侧项目列表

### 3.1 功能

左侧用于管理所有项目。

初步功能包括：

- 显示项目列表
- 新建项目
- 重命名项目
- 删除项目，后续再做，第一版可以先隐藏或只做软删除
- 搜索项目
- 按状态筛选项目，后续扩展

### 3.2 新建项目逻辑

新建项目时需要完成两件事：

1. 在 SQLite 中创建一条项目记录。
2. 在本地项目根目录下创建一个对应的项目文件夹。

示例：

```text
ProjectWorkspace/
└── 项目A/
    ├── 任务分配与完成情况/
    ├── 节点提醒/
    ├── 论文-专利/
    ├── 项目周会记录/
    ├── 项目文档/
    ├── 项目测试/
    ├── 设备-IP-流片/
    ├── 财务情况更新/
    ├── 人员信息/
    ├── 原始文件/
    └── 知识库/
```

### 3.3 重命名项目逻辑

重命名项目时需要同步修改：

- SQLite 中的项目名称
- 本地项目目录名称

需要注意：

- 如果目录重命名失败，数据库不应单独成功修改。
- 如果新名称已经存在，需要提示用户换一个名称。
- 后续可以增加项目编号，避免中文名称变化导致路径长期不稳定。

## 4. 右侧上方：项目基本信息

右侧上方展示当前项目的基本信息。

初步字段包括：

| 字段 | 说明 |
|---|---|
| 项目名称 | 当前项目名称 |
| 项目来源 | 如国家项目、横向项目、企业合作、自研项目等 |
| 开始时间 | 项目启动日期 |
| 结束时间 | 后续可选 |
| 项目经费 | 项目预算或到账经费 |
| 合作单位 | 参与单位或合作企业 |
| 项目负责人 | 后续建议加入 |
| 财务负责人 | 项目财务对接与经费更新负责人 |

建议第一版以 SQLite 为主，后续增加：

- 从 Excel 导入项目基本信息
- 导出项目基本信息为 Excel
- 批量更新项目信息

## 5. 右侧中部：项目标签页

每个项目包含多个标签页。

第一版建议包含：

- 任务分配与完成情况
- 节点提醒
- 论文/专利
- 项目周会记录
- 项目文档

- 财务情况更新

标签页的作用：

### 7.1 跨平台同步约束

使用同步软件时，第一版建议遵守以下规则：

- 同一时间只在一台电脑上编辑同一个工作区。
- 不建议多台电脑同时打开并修改同一个 `app.sqlite`。
- App 启动时检查是否存在同步软件生成的冲突文件。
- 每次启动或关键写入前，可以自动备份 `app.sqlite` 到 `backups/`。
- 数据库中只保存相对路径，例如 `projects/project-0001-项目A/原始文件/任务书.pdf`。
- 文件名尽量避免系统保留字符，例如 `\ / : * ? " < > |`。

### 7.2 FastAPI 文件访问方式

浏览器不能直接自由访问本地文件系统，因此需要通过 FastAPI 提供文件接口。

建议接口能力：

- 上传文件到指定项目标签目录
- 删除指定文件
- 重命名文件或修改文件备注
- 下载文件
- 在浏览器中预览 PDF
- 调用系统默认应用打开文件，后续可选

第一版可以优先支持 PDF 预览和文件下载。

再后续可以把前端静态文件打包后交给 FastAPI 托管，实现只启动一个服务：

```text
http://127.0.0.1:8000
```

## 11. 技术路线建议

第一阶段推荐：

- 后端框架：FastAPI
- 前端：React + TypeScript
- 本地数据库：SQLite
